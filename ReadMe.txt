在面向对象设计模式中存在的是五大原则和一个法则。
 
>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>
五大原则:

 单一职责 SRP(Single Responsibilities Principle)
 	每个类或者模块应该只有一个可以改变的动机。
 开闭原则 OCP(Open Close Principle)
 	开闭原则即为对原有的代码扩展开放，修改关闭。
 里氏代换 LSK(Liskov Substitution Principle)
 	里氏替换即为所有引用父类的地方都可以使用子类来替代。
 依赖倒置 DIP(Dependence Inversion Principle)
 	所谓依赖倒置就是面向超类编程，超类可以为接口，也可以为抽象类。
 接口隔离 ISP(Interface Segregation Principle)
 	一个类对另外一个类的依赖应该遵循专业的小接口（单一职责。。），而不应该是大接口
>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>
一个法则是：

迪米特 DP (Demeter Principle)
 	迪米特法则是管理中涉及的一个术语，被引用到面向对象术语体系中，又称知识最少原则。
